# MRDI Smart Fab Portal · 测试计划

> 版本：v1.0 (Demo Sprint 1) · 文档类型：测试计划 · 范围：全 7 个页面

---

## 一、概述

### 1.1 项目背景

**MRDI Smart Fab Portal** 是面向半导体晶圆厂的统一工作门户，整合 CIM-RMS（需求工作流）、CIM-IMS（报案处理）、MDM（主数据管理）、SPC 趋势、告警中心、班次交接等业务模块，提供一屏全厂健康度总览。

### 1.2 测试目标

- 验证 Portal 7 个页面功能完整性
- 确保品牌色系 / 布局 / 交互符合 MRDI 设计规范
- 保障 SPA 路由跳转、各组件交互状态正确
- 验证 Mock 数据渲染与图表展示

### 1.3 测试范围

| 页面 | 路径 | 类型 | 范围 |
|------|------|------|------|
| Dashboard | `/` | 功能 + 界面 | KPI Strip、产线矩阵、Lot 甘特图、告警 Top 5、子系统卡片 |
| CIM-RMS 入口 | `/cimrms` | 功能 + 界面 | 统计卡片、最近活动列表、外部跳转 |
| CIM-IMS 入口 | `/cimims` | 功能 + 界面 | 统计卡片、最近事件列表、外部跳转 |
| MDM 入口 | `/mdm` | 功能 + 界面 | 注意事项横幅、统计卡片、功能卡片、外部跳转 |
| SPC 趋势 | `/spc` | 功能 + 界面 | 过滤器、SPC 控制图、规则违反列表、导出按钮 |
| 告警中心 | `/alarms` | 功能 + 界面 | Tab 切换、搜索、告警表格、确认/关闭操作 |
| 班次交接 | `/handover` | 功能 + 界面 | 本班小结、待办列表、新增事项、交接确认流程 |

**不在范围（Sprint 1 Demo）**：
- 后端 API 集成（当前全部 Mock 数据）
- 权限 / RBAC
- 多语言
- 性能 / 安全测试

## 二、测试方案

### 2.1 测试类型

| 类型 | 覆盖范围 |
|------|---------|
| **功能测试** | 页面渲染、用户交互、路由跳转、组件状态变更 |
| **界面 / 样式测试** | 品牌色系（MRDI 色板）、布局结构、响应式 |
| **路由测试** | SPA 导航、无效路径跳转 |
| **数据渲染测试** | Mock 数据展示、图表渲染、空数据兜底 |

### 2.2 测试方法

- **手动测试**：交互逻辑（按钮点击、Tab 切换、下拉选择）
- **代码审查**：组件逻辑（Mock 数据结构、状态管理、Recharts 配置）
- **浏览器兼容性**：Chrome / Edge 最新版（根据 PRD）

### 2.3 测试环境

| 项 | 说明 |
|----|------|
| 操作系统 | Windows 11 |
| 测试浏览器 | Chrome / Edge 最新版 |
| Node.js | ≥18（参考项目 `package.json`） |
| 测试启动命令 | `npm run dev` → `http://localhost:5173` |
| 技术栈 | React 19 + Vite 8 + TypeScript 6 + Tailwind 3 |

## 四、风险评估

| 风险 | 等级 | 应对措施 |
|------|------|---------|
| Mock 数据结构与真实 API 不一致 | 中 | 在测试报告中标注字段映射关系，便于后续替换 |
| Recharts 图表在 SSR 场景报错 | 低 | 项目为纯 SPA，无需 SSR |
| 外部子系统 URL 不可达 | 低 | 使用 `localhost:8089` 开箱即用；若子系统未部署则跳转 404，属于预期行为 |
| 样式定制色（如 `bg-ignite`）未在 Tailwind 配置 | 中 | 检查 `tailwind.config.js` 是否注册了自定义颜色类 |
